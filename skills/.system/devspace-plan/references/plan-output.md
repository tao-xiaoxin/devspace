# Plan Writing Checklist

Before persisting a Plan, verify all of the following:

- The title names the actual user outcome, not a generic activity.
- Existing-state claims are backed by files, tests, or configuration that were inspected.
- Scope includes explicit exclusions where likely misunderstandings exist.
- Each action item names a module, interface, behavior, or test target.
- Validation is executable or observable.
- Risks identify a real failure mode, migration concern, security concern, or rollback path.
- The response does not promise edits while Plan Mode is active.
